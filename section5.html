git & github

c'est quoi git ?

Git est un logiciel de versioning créé en 2005 par Linus Torvalds, le créateur de Linux. Il permet de conserver un historique des modifications effectuées sur un projet, ce qui facilite grandement la gestion de projets et permet de travailler en équipe de manière beaucoup plus efficace. En tant que leader incontesté des logiciels de gestion de versions, Git est indispensable pour tout développeur. Les systèmes de gestion de version sont devenus incontournables aujourd'hui en raison de leur capacité à faciliter la gestion de projets.

À quoi ça sert ?

Le système de gestion de version est utilisé pour faciliter la collaboration entre plusieurs développeurs travaillant sur un même projet. Il permet de coordonner le travail en conservant l'historique des modifications effectuées sur des fichiers et en permettant à différentes versions d'un même fichier de coexisterGit est un exemple de logiciel de versioning avec de nombreuses fonctionnalités supplémentaires très utiles. Voici quelques exemples concrets d'utilisation d'un système de gestion de version : _Développement de logiciels. _Gestion de contenu Web. _Collaboration sur des projets de documentation.

Modèle centralisé vs modèle décentralisé

Il existe deux modèles de logiciels de gestion de version : le modèle centralisé et le modèle décentralisé. Dans le modèle centralisé, le code source est hébergé sur un serveur distant central, tandis que dans le modèle décentralisé, chaque utilisateur héberge l'intégralité du code source du projet sur sa propre machine. Le modèle décentralisé a été popularisé par Git et présente plusieurs avantages, notamment la simplicité et la flexibilité du travail, ainsi qu'une sécurité accrue grâce à la possibilité d'utiliser la copie du projet d'un utilisateur comme back-up en cas de corruption du serveur central

C'est quoi Github ?

Plateforme de collaboration et de partage pour les projets de développement de logiciels. GitHub permet aux développeurs de travailler ensemble sur des projets, de publier et de partager des codes sources, de signaler des problèmes et de collaborer pour trouver des solutions. Il offre également des fonctionnalités telles que la gestion des problèmes, le suivi des problèmes, la documentation, les intégrations et la gestion des autorisations pour aider les équipes de développement à travailler efficacem

Les états des fichiers

Dans Git, un fichier peut être soit "suivi" s'il appartient au dernier instantané capturé, soit "non suivi" s'il n'a pas été indexé. Lors du démarrage d'un dépôt, tous les fichiers sont non suivis et doivent être indexés et validés. Les fichiers suivis peuvent avoir l'état "modifié", "indexé" ou "validé". Un fichier est considéré comme "modifié" tant qu'il n'a pas été indexé, et il est indexé lorsqu'on indique à Git qu'il doit faire partie du prochain instantané. Lorsqu'on prend l'instantané, les fichiers faisant partie de celui-ci sont considérés comme validés et le cycle recommence.

La gestion des informations selon Git

Parlons du fonctionnement de Git en tant que système de gestion de version qui considère les données sous forme d'instantanés ou "snapshots". À chaque enregistrement de l'état d'un projet dans Git, il crée un instantané du contenu de l'espace de travail et stocke une référence à cet instantané dans une base de données locale. Cette base de données permet de disposer de l'historique complet d'un projet localement, ce qui rend la plupart des opérations de Git rapides et faciles à réaliser sans avoir besoin d'être connecté à un serveur central distant.
